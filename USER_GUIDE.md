# 📖 阅伴使用指南

欢迎使用 阅伴！本指南将帮助你快速上手所有功能。

---

## 🚀 快速开始

### 1. 启动应用
```bash
cd D:\AIcode\ai-reader
streamlit run app.py
```

### 2. 配置 API Key
首次使用需配置 DeepSeek API Key：
- 在项目目录创建 `.env` 文件
- 添加：`DEEPSEEK_API_KEY=sk-你的key`
- 或在侧边栏直接输入

### 3. 上传第一本书
- 点击"Add a new book"展开上传区
- 选择 PDF 或 TXT 文件
- 可自定义书名（留空自动识别）
- 点击"Add to Bookshelf"

## 📖 阅读界面

#### 🔍 搜索和筛选

底部三个按钮：
- **← Previous Chapter**：上一章

## ⭐ 核心功能详解

### 1. 章节状态自动管理

#### 自动标记规则：
- 首次点击章节 → 自动标记为"📖 正在阅读"
- 切换到其他章节 → 前一章自动标记为"✅ 已完成"
- 点击"Mark as Completed" → 手动标记完成

#### 使用场景：
- **快速跳过不感兴趣的章节**：直接切换，自动标记完成
- **重点章节反复阅读**：手动标记完成后仍可再次进入
- **追踪阅读进度**：一眼看出哪些章节需要回顾

### 2. 书签功能

#### 如何使用：
1. 点击章节右侧的 ☆ 图标添加书签
2. 再次点击 ⭐ 取消书签
3. 勾选顶部 ⭐ 筛选框仅显示书签章节

#### 使用场景：
- 标记重点章节（核心概念、难点）
- 收藏精彩片段
- 快速定位需要复习的内容

### 3. 目录搜索

#### 如何使用：
- 在侧边栏顶部搜索框输入关键词
- 实时过滤匹配的章节标题
- 大小写不敏感

#### 使用场景：
- 长篇书籍快速定位特定主题
- 查找包含关键词的章节
- 结合书签筛选精准查找

### 4. 多级章节

#### 支持的标题格式：
**一级标题**：
- 第X章、第X部、第X篇
- Chapter X / CHAPTER X

**二级标题**：
- 第X节
- 数字编号（1.1、1.2、2.3）
- Section X / SECTION X

#### 显示效果：
- 当前章节下会自动展开子章节
- 子章节以 └─ 符号缩进显示
- 点击主章节切换，子章节内容包含在内

### 5. 智能分段

#### 触发条件：
- 上传的书籍无明确章节标记

#### 分段规则：
- 按双换行识别段落边界
- 每5000字符作为一个自动章节
- 保持段落完整性
- 自动命名为"第X部分"

## 🔧 常见问题

### Q: 章节识别不准确怎么办？
A: 支持以下格式：
- 第X章、Chapter X、CHAPTER X
- 第X节、Section X
- 如果仍无法识别，会自动按5000字分段

### Q: 可以修改章节状态吗？
A: 可以！点击"✅ Mark as Completed"手动标记

### Q: 书签有数量限制吗？
A: 没有限制，可以随意添加

### Q: 搜索支持模糊匹配吗？
A: 支持部分匹配，只要章节标题包含关键词即可

### Q: 对话历史会保存吗？
A: 会！每次切换章节时自动保存，下次进入可继续

### Q: 删除书籍后能恢复吗？
A: 不能，删除是永久性的，请谨慎操作

### Q: 支持什么文件格式？
A: 目前支持 PDF 和 TXT

## 📊 数据存储

### 文件结构
```
shelf/
├── index.json              # 书架索引
└── [book_id]/
    ├── original.pdf        # 原始文件
    ├── full_text.txt       # 提取的纯文本
    ├── chapters.json       # 章节数据（状态、书签）
    └── history.json        # 对话历史
```

### 数据安全
- 所有数据存储在本地
- 对话历史自动保存
- 关闭应用不会丢失进度
